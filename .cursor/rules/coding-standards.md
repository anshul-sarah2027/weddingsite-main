# Coding Standards

## Language

Always TypeScript.

Never use any.

Prefer interfaces.

Strong typing everywhere.

---

# Components

One responsibility per component.

Prefer composition.

Avoid components larger than 200 lines.

Extract reusable parts early.

---

# Styling

Tailwind first.

No inline styles.

No hardcoded spacing.

Use design tokens.

---

# Naming

PascalCase

Components

camelCase

Functions

Hooks

UPPER_CASE

Constants

---

# Imports

External

↓

Internal

↓

Relative

Group imports cleanly.

---

# Folder Naming

lowercase

kebab-case

Consistent everywhere.

---

# React

Prefer Server Components.

Use Client Components only when required.

Keep state local.

Avoid unnecessary context.

---

# Performance

Use next/image.

Lazy load heavy sections.

Dynamic import when appropriate.

Avoid unnecessary rerenders.

---

# Accessibility

Semantic HTML.

Proper heading order.

Keyboard support.

ARIA labels.

Visible focus states.

Reduced motion support.

---

# SEO

Metadata.

Open Graph.

Twitter cards.

Structured data.

Sitemap.

Semantic HTML.

---

# Git

Small commits.

Clear commit messages.

Example

feat: add arrival hero section

fix: improve navbar spacing

refactor: extract timeline component

---

# General Rule

Readable code is more important than clever code.

Future developers should understand the project within minutes.
